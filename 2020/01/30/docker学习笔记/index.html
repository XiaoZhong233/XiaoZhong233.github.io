查看容器日志

docker logs -f -t --tail 10 [容器ID]

-t 是加入时间戳 time

-f 跟随最新的日志打印 follow

–tail 数字 显示最后几条

查看容器内进程

docker top [容器ID]

查看容器内部细节

docker inspect [容器ID]

进入正在运行的容器

docker attach [容器ID]

不进入但执行某个操作

docker exec -t [容器ID] ls -l /tmp

docker exec -t [容器ID] /bin/bash

容器数据卷

当我们在使用docker容器的时候，会产生一系列的数据文件，这些数据文件在我们关闭docker容器时是会消失的，但是其中产生的部分内容我们是希望能够把它给保存起来另作用途的，Docker将应用与运行环境打包成容器发布，我们希望在运行过程钟产生的部分数据是可以持久化的的，而且容器之间我们希望能够实现数据共享。

通俗地来说，docker容器数据卷可以看成使我们生活中常用的u盘，它存在于一个或多个的容器中，由docker挂载到容器，但不属于联合文件系统，Docker不会在容器删除时删除其挂载的数据卷。特点：

  1. 数据卷可以在容器之间共享或重用数据

  2. 数据卷中的更改可以直接生效

  3. 数据卷中的更改不会包含在镜像的更新中

  4. 数据卷的生命周期一直持续到没有容器使用它为止

数据卷

命令添加

docker run -it -v /宿主机绝对路径目录:/容器内目录 镜像名

如果需要设置只读只需要在路径后加入:ro即可

例如：

docker run -it -v /宿主路径:/容器路径:ro 镜像名

在dockerfile中使用VULUME指令

docker build 构建新的模板

启动镜像容器

docker run -it zhw/centos

可以发现自动挂载了数据卷了哦

通过docker inspect查看详细信息，发现在宿主机的/var/lib/docker/volumes下放置着数据卷

这样就添加成功了。

数据卷容器

容器挂载数据卷，其他容器通过挂载这个（父容器）容器实现数据共享，这个挂载数据卷的容器，就称为数据卷容器。

数据卷容器的对象是容器与容器之间，数据卷的对象是宿主与容器之间。

例子

在上述基础上，在dockerfile中添加好数据卷。

先启动一个父容器dc01，并修改dataVolumeContainer2的内容。

启动父容器dc01

[root@localhost /]# docker run -it --name dc01 zhw/centos

修改

创建子容器dc02并继承父容器dc01

docker run -it --name dc02 --volumes-from dc01 zhw/centos

查看是否有dc01创建的文件

结果是有的。

同理，创建一个dc03的子容器也继承dc01

docker run -it --name dc03 --volumes-from dc01 zhw/centos

同样也是存在dc01创建的dc01_add.txt文件的。

经过实验

父修改，子也会修改。

子修改，父也会修改。除非指定了:ro参数

子删除，依然有文件。

综上，数据卷的生命周期一直持续到没有容器使用它为止

也就是只要在某个子容器上存在该数据，数据就不会消失。
